# cterm — Neovim + Claude Code TUI Playground

**Date**: 2026-04-27
**Status**: Approved
**Owner**: anton.isoaho@hogia.se

## Purpose

A self-contained, reproducible Neovim playground for experimentation, paired
with Claude Code in a side-by-side terminal layout. Acts as a sandbox config —
fully isolated from the user's primary Neovim setup — so plugins, keymaps, and
themes can be tried without risk to the daily-driver environment.

The playground also serves as a daily workspace: file explorer + git diff
on the right (Neovim), interactive AI assistant on the left (Claude Code),
inside a single terminal window.

## Success Criteria

1. Launching `cterm.cmd` opens one WezTerm window with two panes: Claude Code on
   the left, Neovim on the right.
2. The Neovim instance uses an isolated config rooted at `C:\git\cterm\nvim`,
   leaving `%LOCALAPPDATA%\nvim` (and any other existing config) untouched.
3. Plugins install on first launch via `lazy.nvim` with no manual steps.
4. The colorscheme matches Claude's warm-orange brand palette
   (primary accent `#D97757`).
5. Built-in git workflow: gutter signs (gitsigns), full diff UI (diffview),
   git status sidebar (neo-tree git source).
6. Standard editor batteries: treesitter, telescope, lualine, bufferline.

## Non-Goals (YAGNI)

- LSP / autocompletion (`mason`, `nvim-lspconfig`, `nvim-cmp`).
- Debugger integration (DAP).
- Session/window persistence beyond what WezTerm provides natively.
- Multi-tab or multi-window WezTerm layouts.
- Cross-platform parity with macOS/Linux beyond a thin bash launcher.

## Architecture

```
+----------------+ +----------------+
|                | |  neo-tree      |
|  claude code   | +----------------+
|  (left pane)   | |  editor + diff |
|                | |  (right pane)  |
+----------------+ +----------------+
        WezTerm window
```

- **WezTerm** is the outer multiplexer (native Windows binary).
- A `gui-startup` event in `wezterm.lua` spawns one window with two panes:
  left runs `claude`, right runs `nvim`.
- A launcher script (`cterm.cmd` on Windows, `cterm` on bash) sets XDG env
  vars so Neovim reads its config from the repo and writes its data into the
  repo's `.data/.state/.cache` directories. This achieves total isolation
  without symlinks or `NVIM_APPNAME` junctions.

### XDG isolation strategy

The launcher exports:

| Env var            | Value                  |
|--------------------|------------------------|
| `XDG_CONFIG_HOME`  | `<repo>`               |
| `XDG_DATA_HOME`    | `<repo>\.data`         |
| `XDG_STATE_HOME`   | `<repo>\.state`        |
| `XDG_CACHE_HOME`   | `<repo>\.cache`        |

Neovim then resolves `stdpath('config')` to `<repo>\nvim`, `stdpath('data')`
to `<repo>\.data\nvim`, etc. `lazy.nvim` installs plugins under
`<repo>\.data\nvim\lazy`. The repo is fully self-contained.

## Components

### Launcher

**`cterm.cmd`** (Windows):

```cmd
@echo off
set "REPO=%~dp0"
set "REPO=%REPO:~0,-1%"
set "XDG_CONFIG_HOME=%REPO%"
set "XDG_DATA_HOME=%REPO%\.data"
set "XDG_STATE_HOME=%REPO%\.state"
set "XDG_CACHE_HOME=%REPO%\.cache"
wezterm start --config-file "%REPO%\wezterm.lua" --always-new-process
```

**`cterm`** (bash) is a mirror with POSIX `export` syntax.

### WezTerm config

`wezterm.lua` registers `gui-startup` to spawn the two panes:

```lua
local wezterm = require('wezterm')
local mux = wezterm.mux

wezterm.on('gui-startup', function(cmd)
  local repo = wezterm.config_dir
  local tab, left, window = mux.spawn_window {
    cwd = repo,
    args = { 'claude' },
  }
  local right = left:split {
    direction = 'Right',
    size = 0.55,
    cwd = repo,
    args = { 'nvim' },
  }
  window:gui_window():maximize()
end)

return {
  color_scheme = 'Gruvbox Dark',
  font = wezterm.font('JetBrains Mono'),
  font_size = 11,
  hide_tab_bar_if_only_one_tab = true,
  window_padding = { left = 4, right = 4, top = 2, bottom = 2 },
}
```

Right pane gets 55% width so the editor has more room.

### Neovim entry

`nvim/init.lua`:

```lua
require('config.options')
require('config.lazy')      -- bootstraps lazy.nvim, loads plugins/*
require('config.keymaps')   -- after plugins so commands exist
```

`config/lazy.lua` clones `folke/lazy.nvim` into `stdpath('data')` on first run
and calls `require('lazy').setup({ { import = 'plugins' } })`.

## Data Flow

1. User runs `cterm.cmd`.
2. Launcher exports XDG env vars and execs `wezterm start --config-file ...`.
3. WezTerm starts, fires `gui-startup`, spawns one window with two panes.
4. Left pane execs `claude`. Right pane execs `nvim`.
5. Neovim resolves config paths via XDG → reads `<repo>\nvim\init.lua`.
6. `init.lua` loads options → bootstraps lazy.nvim → installs plugins on first
   run → loads keymaps.
7. neo-tree opens automatically on startup (configured `auto_open`).
8. User edits files in right pane; gitsigns updates gutter live.
9. `:DiffviewOpen` shows full diff overlay; `q` closes.
10. Claude pane is independent; user copies code/output between panes manually.

## Error Handling

| Failure                              | Behavior                                                                  |
|--------------------------------------|---------------------------------------------------------------------------|
| `wezterm` not on PATH                | Launcher exits with cmd's "not recognized" error.                         |
| `claude` not on PATH                 | Left pane prints error and stays open. User edits `wezterm.lua` to fix.   |
| First launch, no internet            | lazy.nvim shows clone failures; user reruns when online.                  |
| Lazy plugin spec error               | lazy UI shows error inline; nvim still usable.                            |
| Repo not a git repo                  | gitsigns/diffview no-op (built-in graceful behavior).                     |
| Stale plugin lockfile                | `:Lazy sync` resolves.                                                    |

## Testing

Manual checklist (no test framework — config repo):

1. Launch via `cterm.cmd` from File Explorer → window opens, two panes,
   claude left + nvim right.
2. First-launch plugin install completes without errors (`:Lazy log` clean).
3. `<leader>e` toggles neo-tree; tree shows current dir with git icons in repo.
4. Edit a tracked file, save → gitsigns gutter shows change marker.
5. `:DiffviewOpen` shows diff UI with file panel populated.
6. `<leader>ff` lists files; `<leader>fg` greps.
7. Dogfood: run inside `C:\git\cterm` itself; ensure `.data` is git-ignored.
8. Close + relaunch: second start uses cached lockfile, no reinstall.

## Risks & Open Questions

- **WezTerm version skew**: `mux.spawn_window`/`pane:split` API has been stable
  since v20220807. Pin documentation reference; warn user to update if old.
- **Claude CLI updates**: if `claude` is reinstalled to a different path, the
  left pane fails. Acceptable — single-line fix in `wezterm.lua`.
- **Future: LSP**: scoped out now, but layout reserves room — adding `mason` +
  `lspconfig` later requires only new plugin files.
